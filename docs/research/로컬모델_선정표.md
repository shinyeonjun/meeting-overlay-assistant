# 로컬 모델 선정표

## 선정 기준

현재 프로젝트는 “회의 중 실시간 보조”와 “회의 후 문서화”를 동시에 만족해야 한다.  
따라서 모델은 하나를 고르는 방식보다, 경로별로 역할을 나눠 선택하는 것이 맞다.

평가 기준:

- 한국어 품질
- 첫 응답 지연
- 실시간 partial 적합성
- 최종 transcript 품질
- 로컬 실행 안정성
- 메모리 / 장치 요구사항

## 현재 채택 조합

| 경로 | 역할 | 현재 채택 |
| --- | --- | --- |
| live partial | 빠른 실시간 자막 | Sherpa-ONNX streaming |
| archive final | 저장/리포트용 최종 전사 | Faster-Whisper GPU |
| report generation | 고정밀 transcript 기반 문서화 | Faster-Whisper GPU + 리포트 파이프라인 |

## 채택 이유

### Sherpa-ONNX

- 현재 재측정 기준:
  - `first_partial = 0.0262s`
  - `first_final = 0.5312s`
  - `RTF = 0.2014`
  - `WER = 0.9429`
  - `CER = 0.9211`
- 즉시성은 매우 우수해 실시간 오버레이에 적합
- 반면 최종 정확도는 낮아서 `archive_final`이나 리포트용 엔진으로는 부적합
- 결론적으로 `Sherpa-ONNX`는 `live partial / fast-final 후보`로만 채택하는 것이 맞다

### Faster-Whisper GPU

- 현재 기준:
  - `WER = 0.1714`
  - `CER = 0.1140`
  - `RTF = 0.2697`
  - 첫 텍스트 지연 `2.8243s`
- 최종 정확도와 처리량이 모두 안정적
- 리포트 생성과 `archive-final` 기준으로 가장 현실적

## 보류한 후보

### AMD Whisper NPU

- 현재 기준:
  - `WER = 0.3429`
  - `CER = 0.1754`
  - `RTF = 1.1494`
  - 첫 텍스트 지연 `14.3790s`
  - Peak RSS `3176.2MB`
- NPU 실험은 했지만, 현재 측정값 기준으로 GPU Faster-Whisper보다 정확도/속도/메모리 모두 불리했다.

### LangChain / LangGraph 기반 이벤트 추출

- 모델 선택 이슈는 아니지만, 이벤트 추출 품질 관점에서 현재는 기존 prompt-based 방식이 더 나았다.

## 현재 판단

- 실시간 STT와 최종 문서 STT는 같은 요구사항이 아니다.
- 따라서 모델 선정도 경로별로 분리해야 한다.
