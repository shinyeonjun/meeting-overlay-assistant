# 디렉토리 맵

이 문서는 코드베이스를 빠르게 탐색하기 위한 파일 단위 안내다.  
“어디를 봐야 하는가”를 중심으로 정리한다.

## 1. 루트

### `backend/`

FastAPI 서버, STT, 이벤트, 리포트, 저장소 코드가 있다.

### `frontend/`

Tauri + Vite 기반 오버레이 UI 코드가 있다.

### `docs/`

현재 구현 문서, 연구 문서, 내부 메모가 있다.

### `tests/`

pytest 기반 테스트, benchmark fixture, 리포트 생성용 샘플 데이터가 있다.

## 2. backend/app

### `main.py`

백엔드 진입점.  
라우터 등록, 앱 초기화, startup/shutdown 흐름이 있다.

### `api/http/`

HTTP / WebSocket 경계 계층.

#### `routes/`

- `sessions.py`: 세션 생성/종료/overview
- `reports.py`: 리포트 생성/조회/재생성
- `events.py`: 이벤트 조회/수정/전이
- `audio_ws.py`: 오디오 WebSocket
- `text_ws.py`: 텍스트 입력 WebSocket
- `health.py`: health / runtime readiness

#### `schemas/`

FastAPI 응답/요청 스키마 정의.

#### `serializers/`

도메인 모델을 API payload로 변환.

#### `dependencies.py`

서비스 인스턴스 조립, 프로필 해석, preload, composition root 역할.

### `core/`

애플리케이션 전역 설정과 정책.

- `config.py`: 환경 변수와 설정
- `audio_source_policy.py`: 입력 소스별 정책
- `media_service_profiles.py`: STT/미디어 프로필 해석
- `ai_service_profiles.py`: LLM/분석 관련 프로필
- `runtime_readiness.py`: 프론트 readiness 표시용 상태

### `domain/models/`

도메인 모델.

- `session.py`
- `utterance.py`
- `meeting_event.py`
- `report.py`
- `screen_context.py`

### `domain/shared/enums.py`

세션, 이벤트, 리포트 등에서 공통으로 쓰는 enum 집합.

### `services/audio/`

오디오와 STT 핵심 경로.

#### `pipeline/`

- `audio_pipeline_service.py`: 실시간 핵심 orchestration
- `stream_alignment_manager.py`: partial/final 정합성 관리
- `live_stream_utterance.py`: live utterance 표현 객체

#### `stt/`

- `hybrid_streaming_speech_to_text_service.py`
- `sherpa_onnx_streaming_speech_to_text_service.py`
- `faster_whisper_speech_to_text_service.py`
- `faster_whisper_streaming_speech_to_text_service.py`
- `speech_to_text_factory.py`
- `transcription.py`

#### `segmentation/`

- `speech_segmenter.py`
- `silero_vad_segmenter.py`

#### `filters/`

- `transcription_guard.py`
- `audio_content_gate.py`

#### `io/`

- `live_audio_capture.py`
- `session_recording.py`
- `wav_utils.py`
- `wav_chunk_reader.py`

#### `preprocessing/`

노이즈 제거/전처리 관련 계층.

### `services/analysis/`

이벤트 추출과 인사이트 분석.

#### `analyzers/`

- `insight_pipeline_meeting_analyzer.py`
- `llm_based_meeting_analyzer.py`
- `rule_based_meeting_analyzer.py`
- `fallback_meeting_analyzer.py`
- `analyzer_factory.py`

#### `llm/`

- `providers/`: Ollama/OpenAI 등 provider 구현
- `clients/`: completion client
- `extraction/`: prompt builder, response parser, spec
- `factories/`: provider/client 생성기
- `contracts/`: 인터페이스와 모델

#### `rules/`

규칙 기반 이벤트 판단 로직.

#### `observability/`

인사이트 관련 지표 모듈.

### `services/events/`

이벤트 저장, 수동 수정, lifecycle 전이.

- `meeting_event_service.py`
- `event_management_service.py`
- `event_lifecycle_service.py`

### `services/reports/`

리포트 생성 전체.

#### `core/`

- `report_service.py`: 리포트 생성/조회/재생성 중심

#### `audio/`

- `audio_postprocessing_service.py`: 녹음 기반 transcript 생성

#### `composition/`

- `markdown_report_builder.py`
- `speaker_event_projection_service.py`
- `simple_pdf_writer.py`

#### `refinement/`

- `structured_markdown_report_refiner.py`
- `llm_markdown_report_refiner.py`
- `report_refiner_factory.py`

### `services/sessions/`

- `session_service.py`
- `session_finalization_service.py`
- `session_overview_service.py`
- `overview_builder.py`
- `topic_summarizer.py`

### `infrastructure/persistence/sqlite/`

SQLite 스키마와 repository 구현.

- `schema.py`
- `database.py`
- `repositories/`

### `repositories/contracts/`

repository 인터페이스 정의.

## 3. frontend/overlay/src

### `main.js`

프론트 초기화와 컨트롤러 연결.

### `controllers/`

기능별 UI orchestration.

- `session-controller.js`: 세션 시작/종료, readiness, 수동 리포트 생성
- `events-controller.js`: 이벤트 수정/처리 완료
- `report-controller.js`: 세션 탭 내 리포트 상태 반영
- `shared-rendering.js`: overview, 상태 배지 등 공통 렌더
- `ui-controller.js`: 창/레이아웃 공통 제어

#### `controllers/live/`

- `live-connection.js`: 오디오 스트림 연결
- `live-payload-handler.js`: payload 수신 후 state 반영
- `live-caption-renderer.js`: 메인 live 자막 렌더링
- `live-feed.js`: 누적 transcript 표시

### `state/`

- `app-state.js`
- `session-store.js`
- `runtime-store.js`
- `live-store.js`
- `events-store.js`

### `services/`

- `api-client.js`: HTTP API 호출
- `live-socket.js`: WebSocket 연결
- `payload-normalizers.js`: 서버 payload 정규화
- `tauri-live-audio.js`: Tauri 브리지
- `web-speech-recognizer.js`: 브라우저 음성 인식 경로

### `renderers/`

- `cards.js`: 이벤트 카드 렌더
- `transcript.js`: transcript 출력

### `dom/`

- `elements.js`: DOM 요소 캐시

### `styles/`

- `app.css`
