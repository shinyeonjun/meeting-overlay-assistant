# 구조 승격 이행

이 문서는 예전 로컬 MVP 구조에서 현재 제품 구조로 옮겨오면서 무엇이 정리됐는지 기록한다.

## 현재 기준

- 공식 런타임은 `server / client / shared / deploy`
- 개발 기본 정본은 PostgreSQL
- retrieval은 `pgvector`를 사용
- 리포트는 `post_meeting pipeline + report_generation_job` 기준

## 이미 정리된 축

- 인증: `email` 기반 식별자 제거, `login_id` 전환
- 권한: `users.role` 제거, `workspace_role` 중심
- 세션: `source` 제거, `primary_input_source` 정리
- participation: `session_participants`, `participant_followups` 분리
- context: account / contact / context_thread 정리
- events: 레거시 필드 제거
- report: `snapshot_markdown`, `report_audit_logs` 제거
- retrieval: `search_text` 제거, `title + body -> search_tsv`
- OCR 축: `screen_contexts` 제거

## 현재 남은 일

- retrieval 품질 튜닝
- 공유 UX 정리
- 오디오 보관 정책 문서화
- 운영/배포 문서 고도화
