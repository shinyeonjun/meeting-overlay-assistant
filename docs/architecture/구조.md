# 프로젝트 구조

> 문서 목적: 현재 운영 기준 저장소 구조와 모듈 책임을 정리한다.  
> 최종 갱신: 2026-03-05

---

## 목표

회의 중에는 `실시간 partial/final 자막 + 이벤트 카드`를 오버레이로 보여주고,  
회의 후에는 같은 세션 데이터를 기반으로 구조화 문서를 생성한다.

## 기능 축 기준 구조

### 회의 중 실시간 STT 파이프라인

| 경로 | 역할 |
|---|---|
| `backend/scripts/stream_live_audio_ws.py` | 시스템 오디오 캡처 후 WebSocket 송신 |
| `backend/app/api/http/routes/audio_ws.py` | 오디오/개발텍스트 WebSocket 엔드포인트 |
| `backend/app/api/http/dependencies.py` | source별 오디오 파이프라인 조립 |
| `backend/app/services/audio/pipeline/audio_pipeline_service.py` | 세그먼트 처리, partial/final 송신, 이벤트 생성 |
| `backend/app/services/audio/pipeline/stream_alignment_manager.py` | `segment_id` 정합성, grace matching, backpressure |
| `backend/app/services/audio/stt/hybrid_streaming_speech_to_text_service.py` | `partial + final` 하이브리드 조합 |
| `backend/app/services/audio/stt/sherpa_onnx_streaming_speech_to_text_service.py` | fast partial 엔진 |
| `backend/app/services/audio/stt/faster_whisper_speech_to_text_service.py` | heavy final 엔진 |
| `frontend/overlay/src/controllers/live/live-connection.js` | 소스별 연결 전략(Web Speech/Tauri/WebSocket) |
| `frontend/overlay/src/services/web-speech-recognizer.js` | mic용 Web Speech API 래퍼 |
| `frontend/overlay/src/controllers/live/live-caption-renderer.js` | `segment_id` 기반 partial→final 교체 렌더링 |

## STT 엔진 전략

| 레이어 | 현재 전략 |
|---|---|
| `mic` | 1순위 Web Speech API, 실패 시 backend STT fallback |
| `system_audio partial` | `sherpa_onnx_streaming` (저지연) |
| `system_audio final` | `faster-whisper` (정확도) |
| `mic_and_audio` | mic(Web Speech) + system_audio(hybrid STT) 병행 |
| `fallback/실험` | moonshine, simulstreaming, sensevoice 경로 유지(기본 비활성) |

관련 설정:
- `backend/config/media_service_profiles.json`
- `backend/config/audio_source_profiles.json`
- `.env` (`STT_BACKEND_*`, `ANALYZER_*`, `REPORT_REFINER_*` 등)

## 프론트엔드 컨트롤러 구조

```text
live-controller.js (파사드)
 └─ controllers/live/
    ├─ live-connection.js
    ├─ live-payload-handler.js
    ├─ live-caption-renderer.js
    └─ live-feed.js
```

핵심:
- `live-controller.js`는 재수출 파사드 역할만 담당한다.
- 실시간 연결/파싱/렌더링 책임은 `controllers/live/*`로 분리되어 있다.

## 실행

백엔드:

```powershell
cd D:\caps
.\venv\Scripts\activate
uvicorn backend.app.main:app --reload
```

오버레이:

```powershell
cd D:\caps\frontend
npm run overlay:tauri:dev
```
