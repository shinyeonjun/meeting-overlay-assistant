# 개발 계획

이 문서는 내부 작업 기록과 우선순위 메모다.

현재 코드 동작의 단일 기준 문서가 아니며, 공식 구조와 정책은 `docs/architecture/`, `docs/product/` 문서를 우선한다.

## 현재 단계 판단

이 프로젝트는 기능 추가 중심 단계에서 품질 개선 단계로 넘어왔다.

이미 닫힌 것:

- 세션 생성/종료
- readiness gating
- 실시간 자막 기본 흐름
- 이벤트 관리 기본 UX
- 수동 리포트 생성
- 세션별 리포트 폴더 구조
- 녹음 기반 고정밀 리포트 파이프라인

현재 남은 핵심 축:

1. STT 품질
2. 이벤트 추출 precision
3. 리포트 분석 품질

## 현재 우선순위

### 1. 리포트 분석 품질

- 중복 제거
- 메타 문장 제거
- 섹션 문장 자연스러움
- PDF 가독성

### 2. 이벤트 추출 precision

- 질문 외 타입 오분류 감소
- evidence 품질 개선
- 최종 리포트 반영 규칙 정교화

### 3. STT 체감 품질

- partial/final 연결감 개선
- stability 기반 UI 개선
- 초기 누락 감소
- endpointing 계층 분리

## 보류 중인 것

- LangChain 이벤트 추출 도입
- LangGraph 도입
- 자동 리포트 생성 복귀
- 과한 UI 기능 확장

보류 이유:
- 현재 병목은 프레임워크가 아니라 품질
- 지금은 도입 이득보다 복잡도 증가가 큼
