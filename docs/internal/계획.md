# 캡스톤 개발 계획

> 프로젝트명: 로컬 AI 회의 오버레이 어시스턴트  
> 목표: 회의 중에는 중요한 맥락을 실시간에 가깝게 잡아주고, 회의 후에는 실행 가능한 문서를 완성한다.  
> 최종 갱신: 2026-03-05

---

## 1. 제품 정의

이 프로젝트는 크게 두 가지 기능으로 구성된다.

1. `회의 중 기능`
- 회의 흐름에서 중요한 질문, 결정, 액션 아이템, 리스크를 빠르게 추출한다.
- 이를 화면 위 오버레이 UI로 보여줘서 회의 진행을 돕는다.

2. `회의 후 기능`
- 회의 중 누적된 이벤트와 발화 근거를 정리한다.
- 회의가 끝난 뒤 구조화된 회의 문서로 자동 변환한다.

핵심 메시지는 다음과 같다.

```text
회의 중에는 중요한 것을 놓치지 않게 돕고,
회의 후에는 바로 공유 가능한 문서를 남긴다.
```

---

## 2. 핵심 목표

### 회의 중 목표

- 오디오와 화면 맥락을 계속 수집한다.
- 발화 단위를 질문, 결정, 액션 아이템, 리스크, 현재 주제로 구조화한다.
- 오버레이 카드에 현재 중요한 것만 빠르게 반영한다.
- 완벽한 자막보다 회의 진행에 실제로 필요한 정보 노출에 집중한다.

### 회의 후 목표

- 회의 전체 흐름을 주제별로 정리한다.
- 질문과 답변, 결정사항, 액션 아이템, 리스크를 문서로 묶는다.
- Markdown, HTML, PDF 형태로 결과물을 만든다.
- 오버레이에서 보인 정보와 최종 문서가 자연스럽게 이어지도록 한다.

### 기술 목표

- 핵심 추론 경로는 로컬 환경에서 동작하도록 설계한다.
- GPU/NPU 역할을 실측 데이터 기반으로 분리한다.
- 실시간 파이프라인과 최종 문서화 파이프라인을 분리해 속도와 정확도를 동시에 맞춘다.
- 입력 소스별로 최적 경로를 분리한다 (`system_audio=로컬 하이브리드 STT`, `mic` 경로는 선택형으로 운용).

---

## 3. 하지 않을 것

- 모든 발화를 완벽한 자막으로 제공하는 것
- 화자 분리와 감정 분석을 MVP에 포함하는 것
- 제스처 해석을 핵심 기능으로 두는 것
- 클라우드 의존형 서비스로 만드는 것
- 회의 내용을 전부 이해하는 범용 AI 비서처럼 포장하는 것

---

## 4. 제품 개념

```text
입력 수집
    |
    v
오디오 / 화면 / OCR 텍스트
    |
    v
회의 이벤트 추출
    |
    +--> 질문
    +--> 결정사항
    +--> 액션 아이템
    +--> 리스크 / 이슈
    +--> 현재 주제
    |
    +--> [회의 중] 실시간 오버레이 카드
    |
    +--> [회의 후] 구조화 문서 생성
```

### 회의 중 사용자 경험

- 상단에는 `현재 주제`를 보여준다.
- 우측에는 `질문`, `결정`, `액션`, `이슈` 카드를 쌓는다.
- 이미 뜬 카드는 새 정보가 오면 업데이트한다.
- 해결된 질문은 접거나 아카이브로 이동한다.
- 회의가 옆길로 새거나 미해결 질문이 남아 있으면 눈에 띄게 알려준다.

### 회의 후 사용자 경험

- 회의 요약
- 주제 흐름
- 질문과 답변
- 결정사항
- 액션 아이템
- 미해결 질문 및 리스크
- 주요 화면 맥락

## 5. 개발 범위

### MVP 범위

1. 시스템 오디오 또는 마이크 입력 수집
2. 발화 단위 분리
3. 질문 / 결정 / 액션 / 이슈 / 현재 주제 추출
4. 회의 중 오버레이 카드 UI
5. 회의 종료 후 구조화 문서 생성
6. 화면 OCR 기반 현재 맥락 보조

### 확장 범위

1. 발표 / 유튜브 / 강의 모드
2. 검색 및 타임라인 탐색
3. Notion / PDF / Markdown 내보내기 고도화
4. 화자 구분, 제스처, 고급 멀티모달 분석

## 6. 단계별 계획

### Phase 1. 문제 재정의 및 문서 정리

목표:
- 프로젝트를 `회의 중 지원 + 회의 후 문서화` 두 축으로 재정의한다.

작업:
- 기존 문서 전면 개편
- 도메인 이벤트 정의
- 상태 전이 정의
- DB 스키마 정리

산출물:
- `계획.md`
- `설계.md`
- `db.md`
- `api.md`

### Phase 2. 입력 계층 안정화

목표:
- 회의 입력을 안정적으로 수집한다.

작업:
- 시스템 오디오 / 마이크 입력 통합
- 화면 캡처 주기 설계
- OCR 추출 파이프라인 추가
- 세션 단위 저장 구조 정리

산출물:
- 오디오 수집 서비스
- 화면 컨텍스트 서비스
- 세션 시작 / 종료 흐름

### Phase 3. 회의 중 기능 구현

목표:
- 발화를 빠르게 구조화해 오버레이에 반영한다.

작업:
- 질문 감지
- 결정사항 감지
- 액션 아이템 감지
- 이슈 / 리스크 감지
- 현재 주제 갱신 로직
- 오버레이 상태 저장 및 UI 반영

산출물:
- `meeting_analyzer_service`
- 이벤트 JSON 스키마
- 카드 업데이트 엔진
- 오버레이 UI 1차

### Phase 4. 회의 후 문서화 구현

목표:
- 회의 종료 시 누적 데이터를 체계적인 문서로 정리한다.

작업:
- 주제별 묶음
- 질문 / 답변 정리
- 결정사항 정리
- 액션 아이템 표 생성
- Markdown / HTML / PDF 출력

산출물:
- 회의 결과 문서
- 타임라인 기반 요약
- 리포트 조회 API

### Phase 5. 통합 및 정확도 보강

목표:
- 회의 중 결과와 회의 후 문서를 일관되게 연결한다.

작업:
- 실시간 파이프라인과 최종 문서화 파이프라인 연결
- OCR 맥락 보강
- 이벤트 중복 제거
- 문서 품질 점검

산출물:
- 통합 파이프라인
- 문서 품질 기준

### Phase 6. 하이브리드 STT 고도화 및 발표 준비

목표:
- 캡스톤 발표에서 보여줄 핵심 데모를 안정화한다.

작업:
- `system_audio` 하이브리드 경로 튜닝 (`partial=sherpa`, `final=faster-whisper`)
- `segment_id` 정합성/깜빡임 개선
- fallback 경로 정리
- 성능 측정
- 데모 시나리오 고정

산출물:
- 성능 비교표
- 발표용 시나리오
- 최종 데모 빌드

## 9. 성공 기준

### 회의 중 기능 기준

- 회의 중 질문 카드가 실시간에 가깝게 뜬다.
- 결정사항과 액션 아이템이 바로 누적된다.
- 현재 주제가 계속 갱신된다.
- 화면에 뜬 정보가 과하지 않고 실제로 회의 진행에 도움 된다.

### 회의 후 기능 기준

- 회의 종료 시 구조화된 문서가 자동 생성된다.
- 문서에 질문, 결정, 액션 아이템, 리스크가 빠짐없이 정리된다.
- 회의 중 오버레이 결과와 회의 후 문서가 일관된다.

### 기술 기준

- 세션 시작 / 종료가 안정적이다.
- 오버레이 UI가 회의 중 끊기지 않는다.
- 로컬 AI 경로와 GPU/NPU 역할 분리를 데이터로 설명 가능하다.

## 11. 현재 진행 상태 (2026-03-05)

- `mic`: `Web Speech API` 운영 중, 실패 시 backend STT(`faster_whisper_streaming`) fallback
- `system_audio`: `hybrid_local_streaming` 적용 완료 (partial=sherpa, final=faster-whisper)
- `segment_id` 기반 partial/final 교체 파이프라인 적용 완료
- 실시간 후보 5개 모델 비교 완료 (faster-whisper / moonshine / simulstreaming / sherpa / sensevoice)
- 현재 집중 과제: `system_audio` final 품질 개선, `standalone_final` 비율 축소
