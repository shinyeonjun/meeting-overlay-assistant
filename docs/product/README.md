# 제품 문서 포털

이 폴더는 CAPS의 사용자 흐름, 기능 요구사항, 현재 UI/UX 기준, 운영 정책을 다루는 제품 문서 모음입니다.

핵심 원칙은 `기술 구조보다 사용자 흐름을 먼저 본다`입니다.  
특히 UI/UX를 단순하고 직관적으로 유지하려면, 화면을 먼저 고치는 대신 `사용자 플로우`와 `정보구조(IA)`를 기준으로 판단해야 합니다.

## 문서별 역할

### [요구사항 / 기능명세](요구사항_기능명세.md)

제품이 어떤 기능을 제공해야 하는지와, 어떤 방향으로 확장되는지를 정리합니다.

### [사용자 플로우 / IA](사용자_플로우_IA.md)

UI를 자르거나 단순화할 때 가장 먼저 보는 기준 문서입니다.

- 사용자가 실제로 밟는 플로우
- 화면에 남겨야 할 정보
- 메인 UX에서 제거하거나 운영 콘솔로 옮겨야 할 정보
- `준비 / 진행 / 정리 / 기록` 기준의 정보구조

### [오버레이 UI/UX 명세](오버레이_UIUX명세.md)

현재 `client/overlay` 구현 기준의 화면 책임, 상호작용, 표시 원칙을 설명합니다.  
전략적 방향은 [사용자 플로우 / IA](사용자_플로우_IA.md)를 기준으로 보고, 이 문서는 그 방향이 현재 화면에 어떻게 반영되는지를 다룹니다.

### [이벤트정책 / 상태전이](이벤트정책_상태전이.md)

질문 카드와 이벤트 보드의 상태 전이 규칙을 정의합니다.

### [운영정책 / 비기능](운영정책_비기능.md)

권한, 로그, 저장, 성능, 운영 관점의 정책을 정리합니다.

## 제품 문서 업데이트 원칙

1. 사용자가 실제로 밟는 흐름이 바뀌면 먼저 [사용자 플로우 / IA](사용자_플로우_IA.md)를 갱신합니다.
2. 화면 배치나 표시 원칙이 바뀌면 [오버레이 UI/UX 명세](오버레이_UIUX명세.md)를 같이 갱신합니다.
3. 이벤트 처리 규칙이 바뀌면 [이벤트정책 / 상태전이](이벤트정책_상태전이.md)를 같이 갱신합니다.
4. 운영 정책, 권한, 저장 기준이 바뀌면 [운영정책 / 비기능](운영정책_비기능.md)를 같이 갱신합니다.
