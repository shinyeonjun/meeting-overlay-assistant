# 운영정책 / 비기능

이 문서는 현재 CAPS의 운영, 품질, 저장 정책을 정리한다.

## 현재 운영 목표

- 실시간성
- 정확성
- 운영 추적 가능성
- 확장 가능성

## 인증 / 권한

- 로그인 식별자는 `login_id`
- 권한 기준은 `workspace_members.workspace_role`
- `mine / all` 범위를 분리한다.

## 저장 정책

- 정본 DB: PostgreSQL
- retrieval 계층: pgvector
- 테스트 격리: SQLite
- 파일 artifact: raw audio, transcript, markdown, pdf

## 리포트 정책

- 세션 종료와 리포트 생성은 분리한다.
- 리포트 생성은 `report_generation_job` 상태로 추적한다.
- 공유는 지원한다.
- 별도 감사 로그 축은 현재 운영 범위에서 제외한다.

## 개인정보 / 민감정보

- 회의 음성, 발화 텍스트, 상대방 정보는 민감도가 높다.
- 한국 개인정보보호법 기준으로 보관 기간과 접근 권한을 분리해 관리해야 한다.
- raw audio는 목적 달성 후 삭제 가능한 구조로 본다.
- transcript와 report는 업무 산출물로 관리한다.

## 품질 정책

- live caption은 즉시성을 우선한다.
- 최종 리포트는 full STT 기반 후처리를 우선한다.
- retrieval 품질은 별도 튜닝 대상이다.

## 향후 도입 후보

- Redis queue / lock / cache
- dedicated worker
- 백업 / 복구 자동화
