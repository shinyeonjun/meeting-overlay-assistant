# 요구사항 / 기능 명세

이 문서는 현재 CAPS가 실제로 제공해야 하는 기능을 코드 기준으로 정리한다.

## 제품 목표

1. 회의 중 실시간 보조
2. 회의 후 정식 리포트 생성
3. account / contact / thread 기반 맥락 추적
4. 과거 회의 retrieval과 다음 회의 준비
5. 사용자 기반 리포트 공유

## 현재 구현 범위

### 인증

- `login_id + password` 로그인
- workspace role 기반 권한 제어

### 세션

- draft 생성
- 시작 / 종료
- account / contact / thread 연결
- 참여자 입력

### live

- `mic`, `system_audio`, `mic_and_audio`, `file`
- live caption
- 실시간 이벤트 추출

### 리포트

- markdown / pdf 생성
- report generation job 상태
- artifact 저장
- 공유

### history / retrieval

- timeline
- carry-over
- retrieval brief
- retrieval search

## 현재 제외 범위

- 별도 감사 로그 제품 기능
- OCR / screen context
- Redis 기반 queue / lock
- 별도 Vector SaaS

## 정책 메모

- 리포트 공유는 유지한다.
- 감사 로그 축은 제거했다.
- 리포트 정본은 파일 artifact 기준이다.
- retrieval은 PostgreSQL + pgvector 기준이다.
