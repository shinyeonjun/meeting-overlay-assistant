# 오버레이 UI/UX 명세

이 문서는 현재 `client/overlay` 구현 기준의 화면 책임과 사용자 상호작용 원칙을 설명합니다.

전략적 방향은 [사용자 플로우 / IA](사용자_플로우_IA.md)를 기준으로 보고, 이 문서는 그 방향을 현재 화면에 어떻게 적용할지 정리하는 역할을 합니다.

공식 클라이언트 경로는 `client/overlay/`입니다.

## 1. 기본 UX 원칙

현재 오버레이 UX는 다음 원칙을 기준으로 유지합니다.

1. `simple is best`를 우선한다.
2. 한 화면에는 한 가지 주요 행동만 남긴다.
3. 시스템 내부 사정보다 사용자의 다음 행동을 먼저 보여준다.
4. 운영자용 정보는 메인 사용자 화면에서 최대한 걷어낸다.
5. 사용자는 `준비 -> 진행 -> 정리 -> 기록` 흐름으로 자연스럽게 이동해야 한다.

## 2. 현재 화면 책임

현재 클라이언트 메인 화면은 아래 네 가지 흐름으로 읽혀야 합니다.

### 준비

회의에 들어가기 전에 맥락을 잡는 단계입니다.

- 세션 만들기
- 회의 제목 입력
- 입력 방식 선택
- 회사 / 상대방 / 업무 흐름 선택
- 참여자 입력
- carry-over 확인

여기서 중요한 것은 `세션을 먼저 준비하고, 라이브는 나중에 시작한다`는 점입니다.

### 진행

MVP에서는 실시간 자막을 보는 단계입니다.

- 회의 시작
- 현재 자막 확인
- 최근 자막 확인
- 회의 종료

이 단계에서는 리포트, 공유, 전체 히스토리를 같이 보여주지 않습니다.
실시간 질문 카드는 MVP 기본 화면에서 제외하고, 추후 실험 기능으로만 검토합니다.

### 정리

회의 종료 후 문서화와 공유를 처리하는 단계입니다.

- 리포트 형식 선택
- 리포트 생성
- 최신 결과 확인
- 공유

### 기록

같은 흐름의 과거 기록을 다시 확인하는 단계입니다.

- 최근 세션
- 최근 리포트
- 공유받은 리포트
- context timeline
- carry-over

## 3. 준비 화면 원칙

준비 화면은 `회의를 바로 시작할 수 있을 만큼만` 입력받아야 합니다.

### 포함할 것

- 회의 제목
- 입력 방식
- 회사 / 상대방 / 업무 흐름
- 참여자
- carry-over 요약
- `세션 만들기`
- `회의 시작`

### 메인 화면에서 숨길 것

- 세션 ID
- 내부 상태 코드
- 상세 readiness 값
- 평균 지연, late final, backpressure 같은 운영 수치

## 4. 진행 화면 원칙

진행 화면은 사용자의 집중을 깨지 않는 것이 가장 중요합니다.

### 포함할 것

- 메인 자막
- 최근 자막
- 회의 종료

### 최소화할 것

- 리포트 생성 UI
- 히스토리 목록
- 질문 카드 / 실시간 이벤트 검토 UI
- 운영 모니터 세부 수치
- 인증 / 권한 / 시스템 설명

### late_final 처리 원칙

- `late_final`은 메인 자막을 덮어쓰지 않습니다.
- 대신 누적 transcript와 후속 정리 흐름에는 반영합니다.

이 원칙은 실시간 UX를 안정적으로 유지하기 위한 기준입니다.

## 5. 정리 화면 원칙

정리 화면은 `문서를 만드는 행위`에 집중해야 합니다.

### 포함할 것

- 리포트 형식
- 생성 상태
- 최신 결과
- 공유 대상과 메모

### 줄일 것

- live caption
- 이벤트 편집 UI
- 입력 장치 설정

## 6. 기록 화면 원칙

기록 화면은 단순한 목록이 아니라 `이어보기 화면`이어야 합니다.

### 포함할 것

- 현재 맥락 필터
- 최근 세션
- 최근 리포트
- 공유받은 리포트
- context timeline
- carry-over

### 강조할 것

- 지난 결정 사항
- 액션 아이템
- 리스크
- 남은 질문

### 메인 화면에서 숨길 것

- 내부 경로
- 저장 위치
- 기술용 ID
- 운영 로그

## 7. 운영 정보 배치 원칙

운영 정보는 `메인 사용자 UX`보다 `운영자 UX`에 더 잘 맞습니다.

다음 정보는 [server-admin](../../scripts/server-admin.ps1) 운영 콘솔로 보내는 것을 기본 원칙으로 합니다.

- doctor 결과
- logs
- settings / profiles
- readiness 상세 진단
- queue delay, late final, backpressure 같은 운영 지표

클라이언트 메인 화면에는 사용자가 바로 행동을 결정하는 데 필요한 최소 신호만 남깁니다.

## 8. 현재 구현 기준의 주요 상태 전이

현재 세션 흐름은 아래와 같습니다.

1. `draft`
2. `running`
3. `ended`

의미는 다음과 같습니다.

- `draft`: 회의 준비가 저장된 상태
- `running`: 라이브 회의가 진행 중인 상태
- `ended`: 회의가 종료되고 정리 단계로 넘어간 상태

이 분리는 `세션 만들기`와 `회의 시작`을 분리하기 위한 현재 UX 기준입니다.

## 9. 다음 UI/UX 개편 기준

다음 개편은 아래 순서로 진행하는 것을 권장합니다.

1. 준비 / 진행 / 정리 / 기록을 더 명확히 분리
2. 운영 정보 제거 또는 축소
3. 기록 화면을 timeline + carry-over 중심으로 단순화
4. 참여자 입력과 contact 연결 흐름을 더 명확히 노출
5. 공유 흐름을 `누구에게 왜 보내는지` 중심으로 단순화
