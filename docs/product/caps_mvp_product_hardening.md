# CAPS MVP 제품 보강 기준

## 포지션

CAPS MVP는 범용 회의록 자동화 도구가 아니다. Zoom, Teams, ClovaNote, Notion AI와 정면으로 붙는 방향은 피한다.

현재 MVP 포지션은 다음으로 고정한다.

- 한국어 회의 중심
- 로컬/사내 artifact 중심 저장
- 근거 발화가 붙은 회의록
- 실시간은 질문 포착 overlay로 제한
- 최종 노트, 회의록, 리포트는 세션 종료 후 후처리로 생성

## 제품 신뢰 기준

회의록은 예쁜 문서보다 먼저 검증 가능한 문서여야 한다. 다음 항목은 자동 생성 결과를 그대로 확정하지 않는다.

- 의결사항
- 액션 아이템
- 리스크
- 주요 질문

각 항목에는 가능한 경우 다음 근거를 같이 남긴다.

- 발화자
- 근거 발화 텍스트
- 근거 구간
- 원본 전사 excerpt

근거 구간을 찾을 수 없는 항목은 숨기지 않고 근거 없는 항목으로 남긴다. 사용자는 export 전 검토해야 한다.

## 검증 데이터 전략

30분 이상 한국어 회의 샘플은 공개 소스만으로 안정적으로 확보하기 어렵다. 이 작업은 별도 차수로 분리한다.

MVP 검증 세트는 세 계층으로 관리한다.

- CI smoke: 짧은 synthetic fixture
- 로컬 QA: 동의 기반 직접 녹음 30분 회의 5~10개
- 텍스트 품질: 공개 회의록 transcript 기반 구조화 평가

로컬 QA 음성은 기본적으로 git에 커밋하지 않는다. 파일 위치는 ignored 경로에 두고, manifest만 남긴다.

## 결과물 품질 기준

사용자가 보는 최종 산출물은 내부 파이프라인보다 중요하다.

- HTML 회의록은 고정 템플릿으로 생성한다.
- Markdown, PDF, HTML, document JSON은 같은 `ReportDocumentV1` 정본에서 나온다.
- API에서 artifact를 열 수 있어야 하며, UI는 로컬 파일 경로를 사용자에게 강요하지 않는다.
- PDF는 ReportLab 렌더러를 우선 사용하고, 환경 문제일 때만 fallback writer를 쓴다.

## 사용자 흐름 기준

사용자는 다음 흐름을 로그 없이 끝낼 수 있어야 한다.

1. 회의 시작
2. 회의 종료
3. 후처리 상태 확인
4. 실패 사유 확인
5. 재시도
6. 리포트 열기
7. 근거 발화 확인
8. Markdown/PDF/HTML export

## 개인정보와 보관 기준

한국 사용 맥락에서는 녹음과 전사에 개인정보보호법 리스크가 있다. MVP는 다음 고지를 제품 기준으로 둔다.

- 회의 녹음 전에 참석자 고지를 완료한다.
- 원본 음성, 전사, 회의록 artifact의 보관 위치를 명확히 한다.
- 삭제 책임자를 정한다.
- 외부 AI 또는 외부 STT로 전송되는 설정을 구분한다.
- 기본 보관은 로컬 artifact 저장소 기준으로 설명한다.

현재 기본 로컬 artifact 위치는 `ARTIFACTS_ROOT_PATH` 설정을 따른다.
